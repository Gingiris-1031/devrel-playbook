# 🎙️ DevRel Playbook — Community, Docs & Events SOP

> How to actually run a Developer Relations function: community architecture (Discord/Slack setup, moderation, engagement loops), documentation strategy, event operations from meetups to hackathon sponsorship, developer advocacy, and — the part everyone skips — measuring DevRel ROI and attributing it to revenue.

## Install

```bash
npx skills add Gingiris-1031/devrel-playbook
# or
clawhub install devrel-playbook
```

## TL;DR in your language

> 🇺🇸 The operating manual for a DevRel role: community, docs, events, advocacy, and metrics that survive a budget review.
>
> 🇨🇳 DevRel 岗位的操作手册：社区架构、文档策略、活动运营（meetup 到黑客松）、开发者布道，以及最容易被跳过的 ROI 度量与营收归因。
>
> 🇯🇵 DevRel職務の実務マニュアル：コミュニティ設計、ドキュメント戦略、イベント運営、アドボカシー、そして予算査定に耐えるROI測定まで。
>
> 🇰🇷 DevRel 직무의 운영 매뉴얼: 커뮤니티 설계, 문서 전략, 이벤트 운영, 개발자 애드보커시, 그리고 예산 심사를 통과하는 ROI 측정.
